1. 핵심 요약

본 재현 실험은 KAIST 페이퍼의 "동적 추론의 인프라 비용"이라는 핵심 메시지가 M3 Pro 로컬 환경에서도 패턴 수준에서 재현 가능한지를 검증했다. 절대 수치 비교가 아니라 형상·관계 비교를 목표로 설계했다.

🎯 가장 중요한 결과 — paper §IV-A.2의 "GPU 54.5% idle"이 거의 정확히 재현됨:
Paper measured ReAct's GPU idle ratio at up to 54.5% (NVIDIA DCGM 기반).
Our reproduction: ReAct mean idle_ratio = 54.98% (formula: 1 - (prefill+decode)/e2e).
이는 decode-RAG 연구의 motivation을 무엇보다 강력하게 지지한다 — Wikipedia tool wait 시간이 GPU를 반 이상 놀게 만드는 구조적 비효율이 분명히 존재함.

8. decode-RAG 연구 관점에서의 시사점

본 재현이 사용자 연구(decode-RAG: decode 중 retrieval prefetch / overlap)에 직접 기여하는 핵심 발견:

① 강력한 motivation 근거 — ReAct idle_ratio 55%:
  • 페이퍼 §IV-A.2 측정값: 54.5% (NVIDIA DCGM)
  • 우리 측정: 54.98% (M3 Pro proxy formula 1 - (prefill+decode)/e2e)
  • 두 측정이 0.5%p 안에 일치 — 우연이 아닌 구조적 패턴
  • "GPU가 도구 응답을 기다리는 시간"이 단일 최대 비효율 → decode-RAG의 사전 prefetch 아이디어가 직접 노릴 수 있는 50% wall-clock 절감 가능성
② Tool execution이 latency 지배 — Fig 5 단일 최대 막대:
ReAct에서 Tool wait 17.7s > Decode 14.7s > Prefill 6.6s.
Wikipedia API 호출 1건당 ~1.2s × ~8회 = ~10s 이상이 순수 네트워크 RTT. decode-RAG가 이 RTT를 decode 중에 hide할 수 있다면 ReAct latency 30-40% 단축 가능.
③ Heavy-tail 분포 (p95/p50 = 3.34) — Fig 7:
Agent workloads의 latency 예측 불가능성을 정량화. tail latency 특성이 분명히 존재 = decode-RAG의 잠재적 효과를 측정할 때 mean보다 p95를 봐야 함을 시사.
④ Test-time scaling 수확체감 — Fig 13:
LATS가 ReAct 대비 1.04× accuracy를 위해 25× wall-clock 비용 지불. 페이퍼 §VI의 "sequential scaling 31×" 패턴과 유사한 형상. decode-RAG는 이 비효율 영역을 더 작은 사이즈로 채울 수 있다는 가설을 검증할 baseline 데이터 확보.

9. 알려진 한계 (재현성 제약)

L1. 하드웨어 차이 — A100 + vLLM (페이퍼) → M3 Pro + llama.cpp (우리). 절대 throughput 약 5-10× 느림. 패턴은 동일하나 시간값 직접 비교 불가.
L2. Llama-3.1-8B Q4_K_M 양자화 — 페이퍼는 FP16. Q4 노이즈가 작은 모델에 누적되는 LATS(193 LLM calls)에서 가장 큰 영향. Reflexion이 ReAct보다 정확도 낮은 것도 이 영향으로 추정.
L3. LATS 표본 3개 — 페이퍼는 50. M3 Pro에서 LATS 1쿼리당 ~17-40분 소요로 시간 제약상 축소. Fig 13의 Spearman ρ FAIL의 직접 원인.
L4. Tool callback 미캡처 — Reflexion·LATS·LLMCompiler의 도구 호출이 LangChain BaseTool.invoke 경로를 거치지 않아 우리 TraceCallbackHandler가 못 잡음. n_tool_calls=0, tool_ms=0는 측정 한계. Fig 8의 tool tokens가 0인 직접 원인.
L5. Wikipedia drift — AgentBench는 live Wikipedia API 호출. 페이퍼(2026-01-07)와 우리 실험(2026-05-26) 사이 4개월 동안 Wikipedia 내용이 변했을 가능성. 동일 질문에 다른 검색 결과 가능.
L6. NVIDIA DCGM 없음 — 페이퍼의 GPU compute utilization %는 NVIDIA 전용 도구. Apple GPU에 등가물 없어 우리 idle_ratio는 wall-clock proxy. 의외로 두 값이 거의 일치(54.5% vs 54.98%)했지만 의미가 정확히 동일하지는 않음.
L7. Serving 실험 미수행 — 페이퍼 Fig 10-12 (Poisson 트래픽, 다중 요청 serving)는 단일 사용자 워크로드만 다룬 본 재현 scope 밖. decode-RAG도 single-request 시나리오부터 검증 가능하므로 영향 적음.

10. 결론

M3 Pro + Llama-3.1-8B Q4 환경에서 페이퍼의 패턴 수준 메시지가 대부분 재현됨:

  • LATS > ReAct LLM 호출 (21× vs 페이퍼 9.2×) — 같은 방향, 더 큰 차이
  • Tool wait가 ReAct latency 지배 (44%) — 페이퍼와 일치
  • GPU idle 55% — 페이퍼 54.5%와 사실상 동일
  • Heavy-tail (p95/p50 = 3.34) — agentic workload의 변동성 재현
  • LATS 정확도>비용 트레이드오프 — 25× cost / 1% accuracy gain (수확체감)

이 재현 결과는 사용자의 decode-RAG 연구 motivation에 직접적인 근거로 활용 가능. 절대 수치 비교가 목적이 아니라 구조적 패턴 재현이 목적이었으므로, ξ-이 차이는 한계지 결함이 아님.
